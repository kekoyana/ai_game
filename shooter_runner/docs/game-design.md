# Shooter Runner - ゲーム設計書

## ゲーム概要

モバイル広告でよく見かける「オートシューティング × ナンバー増殖ランナー」ゲーム。
プレイヤーキャラクターが画面奥に向かって自動で前進し、銃を自動連射しながら進む。
プレイヤーは左右のスワイプ（ドラッグ）操作のみでキャラクターを動かし、
道中の「+1」「x2」などのゲートをくぐって数（＝弾の威力 or 仲間の数）を増やし、
押し寄せる敵を倒していく。

```
  ┌─────────────────────────────┐
  │        敵 敵 敵              │  ← 画面奥から敵が出現
  │          ↓                   │
  │   [+3]       [x2]          │  ← 数値ゲート（通過で数が変化）
  │          ↓                   │
  │     ·  ·  ·  ·              │  ← 弾（自動発射）
  │          ↓                   │
  │        【15】               │  ← プレイヤー（現在の数値を表示）
  │     ← ドラッグで左右移動 →   │
  └─────────────────────────────┘
```

## コアメカニクス

### 1. プレイヤー操作
- **移動**: 画面をドラッグ（スワイプ）して左右に移動
- **攻撃**: 完全オート。常に画面奥方向へ弾を連射
- **前進**: 自動。常に一定速度で画面奥へ進む

### 2. 数値システム
- プレイヤーは「数値」を持つ（初期値: 1）
- この数値が攻撃力（弾のダメージ）を表す
- 数値ゲートを通過すると数値が変化する:
  - `+N` : 数値に N を加算（例: 10 → 10+3 = 13）
  - `-N` : 数値から N を減算（例: 10 → 10-2 = 8）
  - `xN` : 数値を N 倍にする（例: 10 → 10x2 = 20）
  - `÷N` : 数値を N で割る（例: 10 → 10÷2 = 5）
- 数値が 0 以下になるとゲームオーバー

### 3. 数値ゲート
- 道中に左右2択のゲートが配置される
- 各ゲートは2つの選択肢を持つ（例: 左=「+5」/ 右=「x2」）
- プレイヤーは左右どちらを通るか選んで数値を変化させる

```
  ┌──────────┬──────────┐
  │   +5     │    x2    │
  └──────────┴──────────┘
       左          右
```

### 4. 敵
- 画面奥から出現し、プレイヤーに向かってくる
- 各敵はHP（体力）を持つ
- 弾が当たるとプレイヤーの数値分のダメージを与える
- 敵がプレイヤーに接触すると、プレイヤーの数値が減少する
- 敵のHPは頭上に数値で表示

### 5. ステージ構成
- ステージは直線の通路
- 一定区間ごとにゲートや敵の配置がある
- ステージ終盤にボス敵が登場（高HP）
- ボスを倒すとステージクリア

## 画面構成

### ゲーム画面（メイン）
```
┌─────────────────────────────┐
│  Stage 1        Score: 1200 │  ← ヘッダーUI
│                             │
│     👾 HP:50    👾 HP:30    │  ← 敵（HPバー付き）
│                             │
│   ┌─[+5]─┬─[x2]─┐         │  ← 数値ゲート
│   └──────┴──────┘          │
│                             │
│     · · · · · · ·          │  ← 弾
│                             │
│        🔫 【25】           │  ← プレイヤー＋現在数値
│                             │
└─────────────────────────────┘
```

### タイトル画面
- ゲームタイトル
- 「TAP TO START」ボタン

### ゲームオーバー画面
- スコア表示
- リトライボタン

### ステージクリア画面
- クリアスコア表示
- 次のステージへボタン

## 視点・描画

- **トップダウン（見下ろし）視点** で2D描画
- プレイヤーは画面下部に固定、背景が上から下にスクロール
- Canvas (HTML5 Canvas API) を使用して描画

## 技術スタック

- **フレームワーク**: React + TypeScript (Vite)
- **描画**: HTML5 Canvas API
- **状態管理**: React hooks (useReducer / useState)
- **入力**: タッチイベント + マウスイベント（PC対応）
- **追加ライブラリ**: なし（ゼロ依存で実装）

## 実装フェーズ

### Phase 1: 基本骨格
- Canvas描画ループ（requestAnimationFrame）
- プレイヤーの左右移動（ドラッグ操作）
- プレイヤーの自動前進（背景スクロール）

### Phase 2: 射撃と敵
- 弾の自動発射と描画
- 敵の出現とHP管理
- 弾と敵の当たり判定
- 敵の撃破演出

### Phase 3: 数値ゲート
- ゲートの配置と描画
- ゲート通過時の数値変化
- 数値の画面表示

### Phase 4: ゲーム性の完成
- ステージ構成（敵・ゲートの配置データ）
- ボス敵
- ゲームオーバー / クリア判定
- スコアシステム

### Phase 5: 演出・UI
- タイトル画面
- ゲームオーバー / クリア画面
- エフェクト（撃破時、ゲート通過時、数値変化時）
- サウンド（任意）
